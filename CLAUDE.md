# ivan-portfolio — стандарт исполнения

Личный сайт-визитка Ивана — витрина его уровня как разработчика. Ставка выше
обычного клиентского проекта: это то, что продаёт его самого, и портфолио
тонкое, поэтому сам сайт обязан быть доказательством уровня.

## Жёсткие правила (без исключений)

1. **После утверждения направления — билд только в реальном проекте, на
   реальных библиотеках** (Three.js/GSAP/Lenis и т.п. через CDN), не в
   Artifact-набросках на голом canvas. Artifact допустим только на этапе
   выбора направления (один экран, для утверждения), не как финальный контент.
2. **Перед показом Ивану — обязательная проверка живьём в браузере**
   (Claude Preview: скриншот + консоль без ошибок + проверка интерактива на
   реальном сервере). "Должно работать" не считается.
3. **Перед показом — adversarial-сверка с реальными референсами.** Зайти на
   живые Awwwards/SOTD-сайты в нужной нише (не по памяти), сравнить, честно
   ответить: это выглядит как черновик или как почти-финал? Черновик — не
   показывать, доделать сначала.
4. **Никаких генерик-паттернов по умолчанию.** eyebrow + headline + subheadline
   + два CTA-баттона + навбар + футер-полоса — под запретом как дефолтный
   скелет, пока не обосновано конкретным содержанием этой страницы.
5. **Фото/референсы — только чистая лицензия.** Unsplash, Pexels, собственные
   скриншоты живых продуктов. Никогда не тянуть картинки напрямую с Pinterest
   (неизвестные лицензии — риск для сайта, который продаёт тебя как профи).

## Контент, который реально есть (не выдумывать сверх этого без запроса)

- **OASIS Lounge** — единственный завершённый кейс, production, живой:
  `oasislounge.org`. Флагманское доказательство.
- **Без личного фото Ивана** — осознанное решение (2026-07-09): сайт продаёт
  через уровень исполнения и реальный кейс, не через лицо.
- Позиционирование: сам сайт — часть портфолио, не только рассказ о работах.

## Стек

Vanilla HTML/CSS/JS, без сборки. Хостинг — GitHub Pages (только статика, без
бэкенда). Форма связи, если понадобится — EmailJS/Formspree (без сервера).
